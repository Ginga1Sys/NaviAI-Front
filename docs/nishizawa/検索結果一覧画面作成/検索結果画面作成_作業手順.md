# 検索結果画面（SCR-04） 作業手順書

## 概要

本手順書は、ダッシュボードから実行された検索クエリを受け取り、検索結果一覧を表示する「検索結果画面（SCR-04）」の実装手順をまとめたものです。

## 前提条件
- 開発環境が整っていること（Node.js / Docker / Docker Compose 等）
- フロントエンドは `NaviAI-Front` を対象とする
- 既存のダッシュボード・投稿一覧画面実装を参照できること

- サイドバー（タグ一覧）コンポーネント: `NaviAI-Front/app/components/SearchSidebar.tsx`

## 実装方針
- ヘッダとサイドバーはダッシュボード画面の既存実装を再利用する。
- ダッシュボードから渡されるクエリパラメータ（例: `q`, `tags`, `page`）を受け取り、初期表示時に検索APIを呼ぶ。
- 検索結果は一覧表示（タイトル、タグ、投稿者、投稿日）とし、クリックで記事詳細ページへ遷移する。
- 画面右側に結果件数とタグ一覧を表示する。タグは検索時にバックエンドから取得したタグ一覧を表示する。
- レスポンスのエラーハンドリングとローディング状態を表示する。

## 詳細な実装手順
1. ファイル・ディレクトリ準備
   - `NaviAI-Front/app/pages` または `NaviAI-Front/app` 下に検索用ページ用ディレクトリを作成する。ファイル名はプロジェクト規約に合わせる。
2. ルーティングとパラメータ受け取り
   - ダッシュボードから遷移される URL（例: `/search_list?q=...&tags=...&page=1`）を想定し、`useSearchParams`（Next.js）等でクエリを受け取る実装を行う。
3. API 呼び出し
   - 検索API（基本設計参照）を呼び出す処理を実装する。既存の `fetcher.ts` を利用する。
   - タグ一覧用APIも同時に呼び、右カラムに反映する。
4. UI 実装
   - 中央カラムに `SearchResultList` を配置し、各記事のタイトル、タグ、投稿者、投稿日を表示するカードを作成する。
   - 右カラムに結果件数と `SearchSidebar`（タグ一覧）を表示する。
   - ヘッダ・サイドバーはダッシュボードと同じコンポーネントを再利用する。
5. 状態管理
   - ローディング、エラー、空結果状態を適切に表示する。
   - ページネーションは必要に応じて UI を用意する（API に合わせる）。
6. スタイル適用
   - 既存の `styles` を参照してレイアウト・見た目を整える。必要ならモジュール CSS を追加する。
7. ドキュメントとディレクトリ構成の更新
   - `NaviAI-Front/docs/nishizawa/NaviAI-Front_ディレクトリ構成.md` に新規ファイルとパスを追記する。
8. 動作確認
   - `NaviAI-Front` 配下で開発サーバを起動し、ブラウザで `/search_list` にアクセスして表示確認を行う。

## コマンド（ローカル確認）
開発用コンテナ／サーバ起動（リポジトリルートが `NaviAI-Front` の場合）:

```bash
cd NaviAI-Front
docker compose -f docker-compose.dev.yml up --build
# またはローカルで npm を使う場合
npm install
npm run dev
```

ブラウザで `http://localhost:3000/search_list` にアクセスして確認する。
