## 実装作業の内容
- 公開トップ画面の実装
- ログイン前に表示する画面なので、実装済みのダッシュボード画面、投稿一覧の画面資産とは異なるUIになる。SCR-12_公開トップ（未ログイン）_イメージ図.svgを参考にして実装を進めること。
- 画面右上のヘッダにログイン画面に遷移するボタンと会員登録画面に遷移するボタンを設定する。
- 画面右にタグ一覧を表示する。タグ一覧はダッシュボード画面と同様にバックエンドのタグ一覧取得APIを呼び出して取得すること。
- 画面中央にはおすすめ記事(未公開)1件と公開記事(2件)を表示する。記事のタイトル、タグ、投稿者、投稿日を表示すること。
- 公開トップ画面の作成でフロントエンドとバックエンドの資産を両方修正する必要があるので、修正も実施する。修正内容は下記
  - バックエンド
    - `NaviAI-Back/app/src/main/java/com/ginga/naviai/config/SecurityConfig.java`
      - `/api/v1/public/**` を `permitAll()` に追加する。
    - `NaviAI-Back/app/src/main/resources/db/migration/*.sql`
      - `knowledge` テーブルへ `visibility` カラムを追加するマイグレーションを追加する。
    - `NaviAI-Back/app/src/main/java/com/ginga/naviai/knowledge/entity/Knowledge.java`
      - `visibility` フィールドを追加する。
    - `NaviAI-Back/app/src/main/java/com/ginga/naviai/knowledge/controller/KnowledgeController.java`
      - 既存 API をそのまま流用しない場合、公開用 controller を新設する。
    - `NaviAI-Back/app/src/main/java/com/ginga/naviai/knowledge/service/KnowledgeServiceImpl.java`
      - 公開用検索条件（`published + public + not deleted`）を扱う処理を追加する。
    - `NaviAI-Back/app/src/main/java/com/ginga/naviai/tags/controller/TagController.java`
      - 既存 API とは別に公開用 controller を新設する、または公開用 service を追加する。
    - `NaviAI-Back/app/src/main/java/com/ginga/naviai/tags/service/TagServiceImpl.java`
      - 公開記事に紐づくタグのみを返す処理を追加する。
    - `NaviAI-Back/app/src/main/java/com/ginga/naviai/tags/repository/TagRepository.java`
      - 公開記事限定のタグ集計クエリを追加する。
  - フロントエンド
    - `NaviAI-Front/app/pages/public-home/index.tsx`
      - ダミーデータ描画をやめ、公開記事・公開タグを API から取得する実装へ変更する。
    - `NaviAI-Front/app/lib/fetcher.ts`
      - 修正は必須ではないが、公開トップ画面では `skipAuth: true` を付けて利用する。
    - `NaviAI-Front/app/` 配下の公開トップ用ルートまたはコンポーネント
      - App Router 優先方針に合わせて、必要に応じて公開トップ画面を `app` 側へ整理する。
- 実装が完了したら、ディレクトリ構成のファイルを更新する。
  - フロントエンドのディレクトリ構成: `NaviAI-Front\docs\nishizawa\NaviAI-Front_ディレクトリ構成.md`
  - バックエンドのディレクトリ構成: `NaviAI-Back\docs\nishizawa\NaviAI-Back_ディレクトリ構成.md`
- 実装が完了したら「NaviAI-Front」フォルダ配下で「docker compose -f docker-compose.dev.yml up --build」を実行し、バックエンドとフロントエンドのdockerコンテナが正常に立ち上がるか確認する。エラーが出て、コンテナの立ち上げに失敗したらログを解析してエラーの原因を解消する。
