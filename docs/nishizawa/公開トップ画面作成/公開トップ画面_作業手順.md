# 公開トップ画面（SCR-12）作業手順書

## 1. 概要
公開トップ画面（未ログイン時に表示）を実装し、公開記事・公開タグをバックエンドAPIから取得して表示する。ヘッダにログイン/会員登録遷移ボタンを設置し、右カラムにタグ一覧、中央におすすめ記事1件と公開記事2件を表示する。

## 2. 前提条件
- リポジトリは最新の feature ブランチを確認済みであること。
- ダッシュボードや投稿一覧の画面資産が既に実装済みであること。
- 調査結果ファイル: docs/nishizawa/公開トップ画面作成/公開トップ画面作成_調査結果.md を参照済み。

- バックエンド: 公開用タグ取得 API（公開記事に紐づくタグのみ返す）

## 5. 実装方針
- 公開トップは認証不要エンドポイントとする。フロントは API 呼び出し時に `skipAuth: true` を付与する。
- 公開記事は `published = true` && `visibility = public` && `deleted = false` を満たすレコードのみ対象とする。
- 既存 API を乱暴に流用せず、公開向けに意図したクエリ（公開限定）を用意する。
- DB 変更は Flyway 形式でマイグレーションファイルを追加する。

## 6. 詳細な実装手順

### 6.1 バックエンド修正（開発者向け）
1. 既存の作成マイグレーション `V3__create_knowledge_and_tag_tables.sql` を修正し、`knowledge` テーブルの `CREATE TABLE` 定義に `visibility` カラムを追加する。
   - 例（変更箇所）: `CREATE TABLE knowledge (..., visibility VARCHAR(32) NOT NULL DEFAULT 'private', ...);`
   - 理由: 初期テーブル定義で対応する方針のため。既に本番で `V3` が適用済みの場合は別途 ALTER マイグレーションを用意してください。
2. `NaviAI-Back/app/src/main/java/com/ginga/naviai/config/SecurityConfig.java` を編集し、公開 API パスを許可する。
   - `/api/v1/public/**` を `permitAll()` に追加。
3. `Knowledge` エンティティに `visibility` フィールドを追加し、マッピングと getter/setter を実装する。
4. 公開用の Service/Controller を追加または既存に公開向けメソッドを追加する。
   - 新規コントローラー候補: `PublicKnowledgeController`（パス `/api/v1/public/knowledge`）
   - 検索条件: `published = true AND visibility = 'public' AND deleted = false` を必須にする。
5. 公開用タグ取得の実装を行う。
   - `TagController`（例: `/api/v1/public/tags`）を新設するか、既存の `TagController` に公開向けエンドポイントを追加する。
   - `TagServiceImpl` に公開記事に紐づくタグのみを返す処理を追加する。
   - 必要に応じて `TagRepository` に公開記事限定の集計クエリを追加する。
   - 例クエリ: `SELECT t.name, COUNT(*) FROM tags t JOIN knowledge_tags kt ON t.id=kt.tag_id JOIN knowledge k ON kt.knowledge_id=k.id WHERE k.published=true AND k.visibility='public' AND k.deleted=false GROUP BY t.name`。
6. 単体テストを追加する（公開 API のレスポンス・認可確認）。

### 6.2 フロントエンド修正（開発者向け）
1. ルートを作成/整理
   - `NaviAI-Front/app/pages/public-home/index.tsx`（または App Router 配下の `app/public-home/page.tsx` に合わせる）を作成。
2. API 呼び出し
   - 記事一覧（公開）とタグ一覧（公開記事に紐づくもの）をそれぞれ `/api/v1/public/knowledge` と `/api/v1/public/tags` から取得する。
   - 既存の `fetcher.ts` を利用し、呼び出し時に `skipAuth: true` を渡す。
3. UI 実装
   - 右上ヘッダに `ログイン` と `会員登録` ボタンを設置（遷移先は既存のルートを使用）。
   - 画面右カラム: タグ一覧（タグ名と件数）
   - 中央: 「おすすめ記事」カード1件、その下に公開記事リスト（最大2件）を表示する。各記事にタイトル、タグ、投稿者、投稿日を表示し、記事クリックで記事詳細へ遷移する。
4. ダミーデータの除去
   - 既に存在するダミーデータレンダリングを API 実データに置き換える。
5. スタイル調整
   - 既存の共通スタイルを再利用しつつ、未ログイン向けのレイアウト（SCR-12 イメージ）に合わせる。

### 6.3 ローカルでのビルド・起動手順
- フロント & バックエンドを Docker Compose で起動（NaviAI-Front 配下で実行）:

```powershell
cd NaviAI-Front
docker compose -f docker-compose.dev.yml up --build
```

- 起動後、ブラウザで公開トップ画面（http://localhost:3000/public-home 或いはルートに応じたパス）を確認する。
