## 修正作業の内容
- ユーザー登録画面で登録ボタンを押して、バックエンドのユーザー登録APIを実行し、APIから正常応答が返却された場合はユーザーが登録に使用したメールアドレスにメールを送信した旨のメッセージを表示する画面をブラウザに表示する。
- 現在のバックエンドのユーザー登録APIのレスポンスの修正が必要か、修正によって他の不具合が発生しないかを確認してからフロントエンドとバックエンドの修正を行う。
- ユーザー登録処理のフロントエンドとバックエンドの資産の修正を両方行いたい。
- ユーザー登録で送信されたメールに記載のURLにアクセスした際に、ユーザー登録完了が正常に実行できれば、登録が完了した旨がユーザーに分かる画面をブラウザに表示する。エラーになった場合は、登録に失敗した旨がユーザーに分かる画面をブラウザに表示する。
- バックエンドのユーザー登録APIのレスポンスを登録完了画面（あるいは登録失敗画面）を表示するURLに変更して、リダイレクトするようにする。
- フロントエンドの資産に登録完了画面と登録失敗画面を追加する。登録完了画面にはログイン画面に遷移するためのリンクを設置する。登録失敗画面にはユーザー登録画面に遷移するためのリンクを設置する。
- フロントエンドの資産に追加するメール送信完了画面、登録完了画面、登録失敗画面のデザイン既存のログイン画面やユーザー登録画面と同様のデザインにする。
- 実装が完了したら、フロントエンドとバックエンドのディレクトリ構成のファイルを両方更新する。
（NaviAI-Front\docs\nishizawa\NaviAI-Front_ディレクトリ構成.md、NaviAI-Back\docs\nishizawa\NaviAI-Back_ディレクトリ構成.md）
- 実装が完了したら「NaviAI-Front」フォルダ配下で「docker compose -f docker-compose.dev.yml up --build」を実行し、ブラウザ上で動作確認を実施する。
