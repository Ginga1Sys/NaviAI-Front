## 実装方針
- 既存のログイン画面実装を踏襲し、UI/背景は共通化する。
- バリデーションはクライアント側で基本チェック（必須、メールドメイン、パスワード長、一致確認）を行い、サーバー側のエラーレスポンスを表示する。
- i18n は今回対応しない（日本語固定）。
- 最低限のアクセシビリティ（label と input の紐付け、aria 属性）を守る。

## 詳細な実装手順
1. ファイル作成
   - `app/register/page.tsx` を作成（Next.js App Router を使用する場合）。
   - UI コンポーネントを `app/components/RegisterForm.tsx` として分離する。
2. フォーム項目
   - 名前 (`name`) — テキスト入力、必須
   - メールアドレス (`email`) — テキスト入力、`@ginga.info` ドメインのみ許可する文言を表示し、クライアント側でドメインチェック
   - パスワード (`password`) — パスワード入力、8文字以上であることを案内
   - パスワード（確認） (`passwordConfirm`) — パスワード一致チェック
   - 「会員登録」ボタン
3. バリデーション（クライアント）
   - 必須チェック: 全入力項目
   - メールドメイン: 正規表現で `^[A-Za-z0-9._%+-]+@ginga\.info$` を満たすこと
   - パスワード長: 8 文字以上
   - パスワード一致: `password === passwordConfirm`
   - 入力エラーは該当フィールドの下に表示する
4. API 呼び出し
   - 送信先: `POST /api/register`（バックエンド設計に合わせる）
   - リクエスト例（JSON）
     ```json
     {
       "name": "山田 太郎",
       "email": "taro@ginga.info",
       "password": "password123"
     }
     ```
   - サンプル呼び出し（`fetch`）
     ```js
     const res = await fetch('/api/register', {
       method: 'POST',
       headers: { 'Content-Type': 'application/json' },
       body: JSON.stringify({ name, email, password })
     });
     const data = await res.json();
     ```
   - 成功時: 登録完了画面へ遷移する、または確認メール送信の案内を表示
   - 失敗時: サーバーからのエラー（バリデーション、重複など）を画面上に表示
5. スタイル
   - ログイン画面と同じ背景・共通ヘッダーを使用する。`app/styles/global.css` や既存のモジュール CSS を流用する。
6. ルーティング
   - `/register` でアクセスできるように `app/register/page.tsx` を配置する。
7. ドキュメント更新
   - 実装完了後、`NaviAI-Front/docs/nishizawa/NaviAI-Front_ディレクトリ構成.md` にファイル/フォルダの追加を反映する。

## 動作確認手順
1. 開発用コンテナを起動（`NaviAI-Front` フォルダ内で実行）:
```powershell
docker compose -f docker-compose.dev.yml up --build
```
2. ブラウザで `http://localhost:3000/register` にアクセスし、画面が表示されることを確認する。
3. 正しい入力で送信し、期待どおりの成功メッセージまたは遷移が発生することを確認する。
4. 異常系（ドメイン違反、短いパスワード、不一致）で適切にエラー表示されることを確認する。
