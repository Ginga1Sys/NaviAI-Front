**概要**
- **目的**: ダッシュボード（SCR-03）と投稿一覧（SCR-05）で共通利用しているヘッダとサイドバーを要件に合わせて修正するための実装手順書。

**前提条件**
- **ローカル開発環境**: `NaviAI-Front` を起点に Docker / Node 環境で動作確認できること。
- **参照実装ファイル**: 以下を編集対象として想定する。
  - [NaviAI-Front/app/components/CommonHeader.tsx](NaviAI-Front/app/components/CommonHeader.tsx)
  - [NaviAI-Front/app/components/Dashboard.tsx](NaviAI-Front/app/components/Dashboard.tsx)
  - [NaviAI-Front/app/my_post_list/page.tsx](NaviAI-Front/app/my_post_list/page.tsx)
  - スタイル: [NaviAI-Front/app/styles/dashboard.module.css](NaviAI-Front/app/styles/dashboard.module.css), [NaviAI-Front/app/styles/global.css](NaviAI-Front/app/styles/global.css), [NaviAI-Front/app/styles/tokens.css](NaviAI-Front/app/styles/tokens.css)

**成果物（想定）**
- `CommonHeader.tsx` のマークアップ・スタイル修正
- サイドバーのマークアップ（既存ファイルまたは `Sidebar` コンポーネント）修正
- 必要に応じた CSS モジュールの追記/更新
- ドキュメント更新: `NaviAI-Front/docs/nishizawa/NaviAI-Front_ディレクトリ構成.md` の該当箇所更新

**実装方針**
- UI 変更は最小限の DOM/スタイル差分で対応する。既存コンポーネントの再利用を優先。
- 画面遷移（管理者パネル、記事投稿）用のルーティングリンクはプレースホルダで実装し、バックエンド実装と合わせて有効化する。
- サイドバーの開閉はクライアント側状態（React state）で実装し、望む場合は `localStorage` に記憶する。

**詳細な実装手順**
1. ヘッダ修正 (`CommonHeader.tsx`)
   - 以下の要素を削除/調整する:
     - 「Ginga」「ダッシュボード」「投稿」のテキストをヘッダから削除する。
     - `+ 新規投稿` ボタンをヘッダから削除する。
   - クイック検索入力欄を左寄せに移動する（親要素のフレックス／グリッド設定を調整）。
   - 影響範囲:
     - `CommonHeader` を直接参照する全ページ（`/dashboard`, `/my_post_list` 等）をビルドして動作確認する。

2. サイドバー修正（既存コンポーネントに実装）
   - まずサイドバーを実装しているファイルを特定（`Dashboard.tsx` 内にインライン実装されている場合はそこを編集、別コンポーネントなら `Sidebar` を編集）。
   - 要件に従ってマークアップを更新:
     - 「ダッシュボード」の上に黒い太字で「Ginga」を表示する（視覚的に最上部に配置）。
     - 「承認」と「投稿一覧」の位置を入れ替える。
     - 「承認」の下に `管理者パネル` を表示し、クリック時に管理者パネル画面へ遷移するためのリンクを設定（暫定ルート例: `/admin`）。
     - `管理者パネル` の下に `+ 新規投稿` ボタンを設置し、クリックで記事投稿画面へ遷移（暫定ルート例: `/posts/new`）。
     - サイドバーの背景色を共通ヘッダと同じ色に変更（`tokens.css` の該当変数を利用するか、共通クラスを適用）。
   - サイドバー開閉機能:
     - トグルボタン（例: ハンバーガーアイコン）を実装し、クリックでサイドバーを閉開できるようにする。
     - 実装は CSS クラス切替 + React state（必要なら `useEffect` で `localStorage` に保存）。

3. スタイル調整
   - `dashboard.module.css` または共通モジュールで以下を調整:
     - クラス設計を見直し、ヘッダとサイドバーで共通色を使う。
     - レスポンシブ対応（狭い画面でサイドバーが自動的に折りたたまれること）。

4. ルーティングとアクセシビリティ
   - `管理者パネル` と `+ 新規投稿` のリンクは暫定パスを設定し、`a` タグや `Link` コンポーネントで遷移可能にする。
   - キーボード操作でサイドバーを開閉できるように `aria-expanded` 等を設定する。

5. 単体確認と差分レビュー
   - 変更をコミットし、差分を確認する。UI 変更はスクリーンショットを取り、レビュー用に添付することを推奨。

6. ドキュメント更新
   - `NaviAI-Front/docs/nishizawa/NaviAI-Front_ディレクトリ構成.md` の該当箇所に、変更したファイルやコンポーネントの一覧と説明を追記する。

**ローカル検証手順**
- `NaviAI-Front` フォルダに移動して以下を実行（PowerShell 例）:

```powershell
docker compose -f docker-compose.dev.yml up --build
```

- ブラウザで以下のページを開き、表示と動作を確認する:
  - http://localhost:3000/dashboard
  - http://localhost:3000/my_post_list

**チェックリスト**
- **コード**: ヘッダのテキスト（`Ginga`, `ダッシュボード`, `投稿`）が削除されている
- **検索入力**: クイック検索入力欄が左寄せになっている
- **新規投稿ボタン**: ヘッダの `+ 新規投稿` が削除され、サイドバーに移動している
- **サイドバー**: `Ginga` が上部に表示されている
- **並び順**: `承認` と `投稿一覧` が入れ替わっている
- **管理者パネル**: `承認` の下に `管理者パネル` が存在し、暫定リンクが設定されている
- **+ 新規投稿**: サイドバーに設置され、暫定リンクが設定されている
- **背景色**: サイドバーの背景色が共通ヘッダと同じになっている
- **開閉**: サイドバーの開閉トグルが動作する
- **レスポンシブ**: モバイル・タブレットで崩れないことを確認
- **ドキュメント**: ディレクトリ構成ファイルを更新

**レビュー項目**
- UI 変更は必ずレビュワーの目で確認してもらう（スクリーンショット添付）。
- ルーティング先（`/admin`, `/posts/new`）を本実装時に確定してからリンク先を更新する。

**備考（実装メモ）**
- 既存のコンポーネント分離状況によっては、`Sidebar` を新規作成して `CommonHeader` と分離することを検討する。分離するとテストや再利用が容易になる。
- 色やトークンは `tokens.css` を優先して使用することでテーマの一貫性を保つ。
