**実装方針（概要）**
- **根本原因特定:** まずブラウザの開発者ツールで API 呼び出し（おすすめ記事・新着記事・ユーザー情報）を確認し、フロントが期待する JSON スキーマとバックエンドの実際のレスポンスに差異がないかを確認する。
- **レンダリング修正:** レスポンスは正常だが表示されない場合、コンポーネント側の props/state の受け取りや条件分岐（存在チェック、配列長チェック）を修正する。
- **管理者表示:** ログイン情報の権限判定ロジックを確認し、管理者ユーザー判定時に「承認待ち」「週次アクティビティ」「サイドバーの承認/管理者パネル」が表示されるように条件分岐を追加・修正する。
- **未ログインリダイレクト:** ページのルーティングガードを実装し、認証トークン/セッションがなければ `/login` へリダイレクトする。
- **ログアウト:** サイドバーの `ログアウト` を追加し、クリック時にバックエンドのログアウト API（例: `POST /api/v1/auth/logout`）を呼び、クライアント側の認証情報を破棄して `/login` へ遷移する。

**詳細実装手順**
1. **再現確認:** ブラウザで `http://localhost:3000/dashboard` を開き、コンソールとネットワークを確認する。
   - **確認項目:** おすすめ記事・新着記事取得 API が呼ばれているか、HTTP ステータス、レスポンス JSON の中身。
2. **API レスポンスの確認:** バックエンドが返すレスポンスのサンプルを確認し、フロントが期待するフィールド名（例: id, title, summary, publishedAt など）と一致するかを検証する。
3. **フロント修正（表示されない問題）:**
   - **対象:** ダッシュボードのリストをレンダリングするコンポーネントを開く。
   - **修正例:** レスポンスが空配列でないかの判定や配列を `.map()` している箇所の null/undefined 安全性を確保する。
   - **期待挙動:** データがある場合はカードを1件（おすすめ）/最大3件（新着）表示する。データ無しの場合は「記事がありません。」を表示する既存挙動は維持する。
4. **フロント修正（管理者表示）:**
   - **対象:** 共通ヘッダー/サイドバーのユーザー権限判定処理。
   - **修正例:** 現在の user オブジェクトに roles または isAdmin 相当の値が含まれているか確認し、存在する場合は管理者専用メニュー（承認/管理者パネル）をレンダリングする。
5. **未ログイン時のルーティングガード実装:**
   - **対象:** `app/layout.tsx` やダッシュボードページのサーバー/クライアントサイド処理。
   - **実装案:** ページ読み込み時に認証トークンをチェックし、未認証なら `router.push('/login')` でリダイレクトする。Next.js の Middleware を使う場合は `/middleware.ts` にルールを追加する。
6. **ログアウト実装:**
   - **UI追加:** サイドバー最下部に `ログアウト` ボタン/リンクを追加する。
   - **API呼び出し:** クリック時にバックエンドのログアウト API を叩き（例: `POST /api/v1/auth/logout`）、クライアント側のトークンを削除する（`localStorage`/`cookie`など）。
   - **遷移:** ログアウト成功後に `/login` へリダイレクトする。
7. **単体確認:** 修正後、以下のパターンで動作を確認する。
   - 管理者ユーザーでログイン → ダッシュボードで「承認待ち」「週次アクティビティ」およびサイドバーの「承認」「管理者パネル」が表示される。
   - 一般ユーザーでログイン → 管理者専用項目が表示されない。
   - 未ログイン状態で `/dashboard` に直接アクセス → `/login` へリダイレクトされる。
   - サイドバー `ログアウト` をクリック → セッションが破棄され `/login` へ遷移する。
