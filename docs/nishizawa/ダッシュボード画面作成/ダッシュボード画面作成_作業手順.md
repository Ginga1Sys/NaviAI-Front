# ダッシュボード画面作成 — 作業手順

## 概要
- 本手順書は、SCR-03（ダッシュボード画面）のフロントエンド実装手順を示します。画面はクイック検索（知識検索）とクイック操作（タグ一覧）を備え、既存APIを呼び出してデータ取得後に別画面へ遷移するまでを実装します。遷移先は今回の実装では仮URLを使用します。

## 表示予定項目
- クイック検索用入力欄（テキスト入力、サジェスト対応）
- クイック操作ボタン（2つ）: `新着` と `おすすめ`
- おすすめ記事（1件表示: タイトル、抜粋、リンク）
- 新着記事（1件表示: タイトル、抜粋、リンク）
- 承認待ち件数（管理者向けバッジ表示）
- 週次アクティビティ（数値要約）
- 画面左のメニュー（ナビゲーション: ダッシュボード、ナレッジ、承認、管理者パネル、新規投稿）

上記項目を画面に配置し、必要に応じてコンポーネントを分離して実装してください。

## 前提条件
- 開発環境がセットアップ済み（Docker / Node / Next.js 等）
- 既存のログイン画面・投稿一覧の画面資産が参照可能であること
- 以下のAPIが利用可能であること:
  - `GET /api/v1/knowledge`（クイック検索用）
  - `GET /api/v1/tags`（タグ一覧取得用）
- バックエンドに新規パッケージを作成する場合の場所（参考）:
  - `NaviAI-Back/app/src/main/java/com/ginga/naviai/tags`

  - スタイル: `NaviAI-Front/app/styles/dashboard.module.css` または components 配下の module CSS

## 実装方針
- 既存の画面資産（ログイン／投稿一覧）を参考にコンポーネント設計を行う。
- プレゼンテーションとデータ取得ロジックを分離する（fetcher / hooks を作成）。
- API呼び出しはクライアント側で行い、取得後に `next/router` の `push()` で仮の遷移先へ移動する。
- スタイルは既存の `tokens.css` / `global.css` を活用し、コンポーネント単位で module CSS を用いる。

## 詳細な実装手順
1. 開発ブランチを切る（例: `feature/dashboard`）。
2. 画面ルートを作成
   - `app/pages/dashboard/page.tsx` または `app/dashboard/page.tsx` を追加し、ラッパーコンポーネントで `Dashboard` をレンダリングする。
3. UIコンポーネント実装
   - `Dashboard.tsx`: レイアウト、ヘッダー、QuickSearch、QuickTags の配置。
   - `QuickSearch.tsx`: 入力フォーム、サジェスト、検索実行で `GET /api/v1/knowledge` を呼ぶ。検索結果選択で仮の詳細ページへ遷移。
   - `QuickTags.tsx`: ページロード時に `GET /api/v1/tags` を呼び、タグ一覧を表示。タグクリックでタグ検索結果ページへ遷移（仮URL）。
4. データ取得実装
   - `fetch` もしくは `fetcher` ユーティリティ（既存があれば流用）を使う。
   - エラーハンドリングとローディング表示を実装する。
5. ルーティング（遷移先は仮）
   - 例: 検索結果へ遷移: `/search?query=...`
   - タグ遷移: `/tags/{tagName}`
6. スタイルの追加
   - `dashboard.module.css` を作り、レスポンシブ対応の基本スタイルを記述する。
7. ドキュメントとディレクトリ構成の更新
   - `NaviAI-Front/docs/nishizawa/NaviAI-Front_ディレクトリ構成.md` に新規ファイルとパスを追記する。
8. ローカルでの起動と動作確認（後述のコマンド参照）。
9. コードレビューを依頼し、フィードバックを反映してPRを作成する。

## サーバ起動と画面確認手順
1. `NaviAI-Front` フォルダへ移動し、開発用コンテナを起動します。Windows の PowerShell 例:

```powershell
cd "d:\AI駆動開発デモ\NaviAI-Front"
docker compose -f docker-compose.dev.yml up --build
```

2. ブラウザで以下にアクセスしてダッシュボード画面を表示確認します:

- http://localhost:3000/dashboard
