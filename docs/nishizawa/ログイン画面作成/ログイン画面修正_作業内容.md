## 実装作業の内容
- ログイン画面資産の修正
- ログイン画面下部にある「アカウント登録はこちら」をクリックすると、アカウント登録画面（SCR-02）に遷移するように実装すること。
- ログイン画面にはメールアドレスを入力するようにする。「または ユーザー名」の部分は削除する。
- ログイン画面にメールアドレスとパスワードを入力してログインボタンをクリックすると、バックエンドのAPI（`POST /api/v1/auth/login`）にリクエストを送信するように実装すること。
- バックエンドのAPIのログインAPIからアクセストークンとリフレッシュトークンを受け取ったら、以下のAPIを実行して、ダッシュボード画面（SCR-03）を表示する。
  - GET /api/v1/users/me
  - GET /api/v1/dashboard
  - GET /api/v1/dashboard/activity
  - GET /api/v1/knowledge（おすすめ記事1件と新着記事3件を取得）
- GET /api/v1/users/me の実行結果から管理者であるかそうでないかが判別できるので、管理者の場合はサイドバーに「承認」と「管理者パネル」のメニュー、ダッシュボード画面の右に管理者向けの「承認待ち」と「週次アクティビティ」を表示する。管理者でない場合はこれらは表示しない。
- 実装が完了したら、ディレクトリ構成のファイルを更新する。
（NaviAI-Front\docs\nishizawa\NaviAI-Front_ディレクトリ構成.md）
- 実装が完了したら「NaviAI-Front」フォルダ配下で「docker compose -f docker-compose.dev.yml up --build」を実行し、ブラウザ上で「http://localhost:3000/dashboard」にアクセスして画面を表示する。
