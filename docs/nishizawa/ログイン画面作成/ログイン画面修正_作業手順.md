**実装方針（要点）**
- メールアドレスのみをログインIDとして受け付ける（「または ユーザー名」削除）。
- ログインは `POST /api/v1/auth/login` を呼び、アクセストークンとリフレッシュトークンを受け取る実装にする。
- トークン受領後、以下を順次取得してダッシュボード表示に必要なデータを準備する:
  - `GET /api/v1/users/me`
  - `GET /api/v1/dashboard`
  - `GET /api/v1/dashboard/activity`
  - `GET /api/v1/knowledge`（おすすめ1件 + 新着3件）
- `users/me` の結果で管理者判定し、UI の表示/非表示を切り替える。
- 実装は既存コードのスタイルに合わせ、小さな差分で済ませる（再利用優先）。

**詳細な実装手順（開発者向け）**
1. 既存ログイン画面の確認
   - `NaviAI-Front/app/login/page.tsx` と `NaviAI-Front/app/components/LoginForm.tsx`（存在する場合）を開く。
2. メールアドレス入力への変更
   - 入力ラベル/プレースホルダを「メールアドレス」に変更。
   - ユーザー名入力や「または ユーザー名」のUI要素を削除。
3. API 呼び出しの実装
   - フォーム送信時に `POST /api/v1/auth/login` を呼ぶ。
   - リクエストボディ例: `{ "email": "...", "password": "..." }`
   - 成功レスポンスで `accessToken` と `refreshToken` を受け取る想定。
4. トークンの保存とヘッダー設定
   - `accessToken` は短期保存（メモリ or `localStorage`）し、認証済みリクエストに `Authorization: Bearer <token>` を付与する。
   - `refreshToken` はセキュアに保存（`httpOnly` クッキー推奨だがフロント側実装では要バックエンド対応）。
   - プロジェクトの既存 `auth` ユーティリティを利用する（`NaviAI-Front/app/lib/auth.ts` 等）。
5. トークン受領後のデータ取得シーケンス
   - 直列または並列で以下を取得（以下の順序を推奨）:
     1. `GET /api/v1/users/me`（管理者フラグ取得）
     2. `GET /api/v1/dashboard`
     3. `GET /api/v1/dashboard/activity`
     4. `GET /api/v1/knowledge`（おすすめ1 + 新着3）
   - UI はローディング表示を出す。エラー時は適切にエラーメッセージを表示。
6. 管理者向けUIの切替
   - `users/me` のレスポンスに `roles` や `isAdmin` がある想定。管理者ならサイドバーに「承認」と「管理者パネル」を表示し、ダッシュボード右に「承認待ち」と「週次アクティビティ」を表示する。
7. アカウント登録画面への遷移
   - ログイン画面下部の「アカウント登録はこちら」をクリックすると `/register`（SCR-02）へ遷移するようにルーティング実装。
8. スタイル調整
   - 既存の CSS モジュール（`NaviAI-Front/app/styles/*` 等）に合わせて修正。
9. ドキュメント更新
   - `NaviAI-Front/docs/nishizawa/NaviAI-Front_ディレクトリ構成.md` を修正し、追加/変更したファイルを反映。
10. コードレビューとマージ
   - PR を作成し、レビュワーに差分の説明を添えてレビュー依頼。

**動作確認手順（ローカル）**
1. `NaviAI-Front` ディレクトリへ移動

```powershell
cd d:/AI駆動開発デモ/NaviAI-Front
```

2. 開発用コンテナ起動（要 Docker）

```powershell
docker compose -f docker-compose.dev.yml up --build
```

3. ブラウザで以下にアクセスして動作確認
- ダッシュボード（ログイン後）: `http://localhost:3000/dashboard`
- ログイン画面（未ログイン）: `http://localhost:3000/login`

4. 手動検証項目
- 正しいメール/パスワードでログインし、ダッシュボードが表示されるか
- 管理者/一般ユーザで表示が切り替わるか
- ネットワークタブで API 呼び出しとトークンの受領を確認する
